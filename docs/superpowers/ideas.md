# Bookworm — Superpower Ideas

- **Annotation-grounded chapter digest** — After reading a chapter, users can request an AI-generated retrospective that synthesizes all of their highlights and notes from that chapter into a coherent summary, with every source annotation shown in the context inspector; this rewards engaged reading and keeps AI output grounded exclusively in the user's own curated selections rather than raw chunk retrieval. Cost: M. Risk: low.
- **Spoiler-safe context retrieval** — Chunk retrieval for AI queries is filtered to passages at or before the user's current reading position, so the AI can never inadvertently reference plot points from unread chapters; the reading-position CFI acts as a hard fence on context assembly and is a pure, unit-testable transform over the existing chunk index. Cost: S. Risk: low.
- **Persistent context pinning** — Users can pin any annotation or passage so it is always included in the AI context window regardless of semantic retrieval results; pinned items are visually distinct in the context inspector, fulfilling the transparency goal and giving readers fine-grained control over what the model knows without any backend involvement. Cost: M. Risk: low.
- **Annotation theme clustering** — An AI call groups all of a book's highlights into 3–5 emergent themes using only the user's own annotation text as input, surfacing patterns in what the reader found noteworthy; the clustering prompt and every annotation fed to the model are shown in the context inspector, keeping the feature fully grounded and transparent. Cost: M. Risk: low.
- **In-context vocabulary lookup** — When a reader selects an unfamiliar word, an AI call receives the surrounding paragraph as context and returns a definition grounded in how the word is actually used in that passage; the full paragraph is shown in the context inspector, keeping the response transparent and anchored to the book's own language rather than a generic dictionary. Cost: S. Risk: low.
- **Active-recall question generation** — After the user finishes a section, an AI call grounded in the passages just read produces 2–3 comprehension questions shown in a collapsible recall panel; each question links back to its source chunk in the context inspector, making the feature fully transparent, grounded exclusively in book text, and implemented as a pure prompt-assembly function over the existing chunk/position system. Cost: M. Risk: low.
- **Reading-gap catch-up digest** — When a reader reopens a book after a configurable absence, an opt-in AI recap grounded exclusively in chunks from already-read chapters (bounded by the existing position-CFI fence) delivers a spoiler-free "story so far" refresh; chunk selection and token-budget trimming are pure, unit-testable transforms that reuse infrastructure already built for spoiler-safe retrieval. Cost: M. Risk: low.
- **Passage-anchored Q&A notes** — Users can pose a question directly attached to any passage; an AI call grounded in that passage and its adjacent chunks generates a response stored as a persistent annotation at the exact CFI location, creating an inline reading-embedded Q&A record without a context-switch to a separate chat view; the full context assembly is shown in the context inspector, and storage reuses the existing annotation model with a typed "qa" variant. Cost: M. Risk: low.
- **Inline passage translation** — When a reader selects text in a foreign-language passage, an AI call receives the selected sentence(s) and the surrounding paragraph as grounded context and returns a translation shown in a floating panel alongside the full context inspector; it directly serves readers of multilingual or academic texts and reuses the existing passage-selection and AI-call infrastructure as a pure prompt-assembly transform with no new libraries. Cost: S. Risk: low.
- **Non-fiction argument outline** — An AI call processes the user's highlights from a non-fiction book in reading order and produces an editable structured outline (thesis, supporting claims, key evidence) stored as a persistent book-level note, with every item linked back to its source highlight in the context inspector; this turns passive underlining into an active synthesis artifact, serves academic and analytical reading workflows, and reuses the existing annotation and AI-call infrastructure without new libraries. Cost: M. Risk: low.
- **Cross-book thematic comparison** — Users select highlights from two different books in their library and send them together as grounded multi-source context for a comparative AI question (e.g. "how do these authors differ on free will?"); context assembly is a pure transform over the existing multi-excerpt infrastructure extended with a per-book provenance label, and every source passage is shown in the context inspector with its book of origin, keeping the feature fully transparent and testable without any new libraries or backend involvement. Cost: M. Risk: low.
- **Entity memory digest** — When a reader wants to recall everything the book has established about a character, place, or concept so far, an AI call grounded exclusively in already-read chunks mentioning that entity (filtered by the existing spoiler-safe CFI fence) assembles a concise "what we know" note; it reuses the chunk-retrieval and position-fence infrastructure as a pure prompt-assembly transform keyed on the entity name, and the full source passages are shown in the context inspector. Cost: S. Risk: low.

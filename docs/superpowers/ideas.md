# Bookworm — Superpower Ideas

- **Annotation-grounded chapter digest** — After reading a chapter, users can request an AI-generated retrospective that synthesizes all of their highlights and notes from that chapter into a coherent summary, with every source annotation shown in the context inspector; this rewards engaged reading and keeps AI output grounded exclusively in the user's own curated selections rather than raw chunk retrieval. Cost: M. Risk: low.
- **Spoiler-safe context retrieval** — Chunk retrieval for AI queries is filtered to passages at or before the user's current reading position, so the AI can never inadvertently reference plot points from unread chapters; the reading-position CFI acts as a hard fence on context assembly and is a pure, unit-testable transform over the existing chunk index. Cost: S. Risk: low.
- **Persistent context pinning** — Users can pin any annotation or passage so it is always included in the AI context window regardless of semantic retrieval results; pinned items are visually distinct in the context inspector, fulfilling the transparency goal and giving readers fine-grained control over what the model knows without any backend involvement. Cost: M. Risk: low.
- **Annotation theme clustering** — An AI call groups all of a book's highlights into 3–5 emergent themes using only the user's own annotation text as input, surfacing patterns in what the reader found noteworthy; the clustering prompt and every annotation fed to the model are shown in the context inspector, keeping the feature fully grounded and transparent. Cost: M. Risk: low.
- **In-context vocabulary lookup** — When a reader selects an unfamiliar word, an AI call receives the surrounding paragraph as context and returns a definition grounded in how the word is actually used in that passage; the full paragraph is shown in the context inspector, keeping the response transparent and anchored to the book's own language rather than a generic dictionary. Cost: S. Risk: low.
- **Active-recall question generation** — After the user finishes a section, an AI call grounded in the passages just read produces 2–3 comprehension questions shown in a collapsible recall panel; each question links back to its source chunk in the context inspector, making the feature fully transparent, grounded exclusively in book text, and implemented as a pure prompt-assembly function over the existing chunk/position system. Cost: M. Risk: low.
- **Reading-gap catch-up digest** — When a reader reopens a book after a configurable absence, an opt-in AI recap grounded exclusively in chunks from already-read chapters (bounded by the existing position-CFI fence) delivers a spoiler-free "story so far" refresh; chunk selection and token-budget trimming are pure, unit-testable transforms that reuse infrastructure already built for spoiler-safe retrieval. Cost: M. Risk: low.
- **Passage-anchored Q&A notes** — Users can pose a question directly attached to any passage; an AI call grounded in that passage and its adjacent chunks generates a response stored as a persistent annotation at the exact CFI location, creating an inline reading-embedded Q&A record without a context-switch to a separate chat view; the full context assembly is shown in the context inspector, and storage reuses the existing annotation model with a typed "qa" variant. Cost: M. Risk: low.
- **Inline passage translation** — When a reader selects text in a foreign-language passage, an AI call receives the selected sentence(s) and the surrounding paragraph as grounded context and returns a translation shown in a floating panel alongside the full context inspector; it directly serves readers of multilingual or academic texts and reuses the existing passage-selection and AI-call infrastructure as a pure prompt-assembly transform with no new libraries. Cost: S. Risk: low.
- **Non-fiction argument outline** — An AI call processes the user's highlights from a non-fiction book in reading order and produces an editable structured outline (thesis, supporting claims, key evidence) stored as a persistent book-level note, with every item linked back to its source highlight in the context inspector; this turns passive underlining into an active synthesis artifact, serves academic and analytical reading workflows, and reuses the existing annotation and AI-call infrastructure without new libraries. Cost: M. Risk: low.
- **Cross-book thematic comparison** — Users select highlights from two different books in their library and send them together as grounded multi-source context for a comparative AI question (e.g. "how do these authors differ on free will?"); context assembly is a pure transform over the existing multi-excerpt infrastructure extended with a per-book provenance label, and every source passage is shown in the context inspector with its book of origin, keeping the feature fully transparent and testable without any new libraries or backend involvement. Cost: M. Risk: low.
- **Entity memory digest** — When a reader wants to recall everything the book has established about a character, place, or concept so far, an AI call grounded exclusively in already-read chunks mentioning that entity (filtered by the existing spoiler-safe CFI fence) assembles a concise "what we know" note; it reuses the chunk-retrieval and position-fence infrastructure as a pure prompt-assembly transform keyed on the entity name, and the full source passages are shown in the context inspector. Cost: S. Risk: low.
- **Annotation-to-argument linker** — Users write a thesis statement and select any subset of their existing annotations to test against it; an AI call grounded exclusively in those chosen annotations evaluates each as supporting, contradicting, or tangential to the thesis and tags it with a relation label, storing the result as a named argument set persisted in the annotation model; unlike the automatic non-fiction outline this feature is wholly user-directed (user picks annotations and writes the thesis), making it a structured reasoning scaffold for research and academic reading while keeping AI grounded in curated user selections shown in full in the context inspector. Cost: M. Risk: low.
- **Passage similarity exploration** — When a reader highlights any passage, a "Find Similar" action queries the existing chunk index with that text as the retrieval key, surfacing the top related passages from elsewhere in the book; an optional AI call grounded exclusively in those retrieved chunks synthesizes how the passages echo, contrast, or develop a shared theme; this turns the Phase 5.2 retrieval engine into a proactive reading tool for tracking motifs and recurring arguments, reusing existing infrastructure as a pure prompt-assembly transform with every source passage visible in the context inspector. Cost: S. Risk: low.
- **Reflective journaling prompter** — After a user finishes any section, an AI call grounded exclusively in the just-read passages generates 2–3 open-ended prompts that invite personal reflection rather than comprehension testing (e.g. "This argument contrasts X and Y — which resonates more with your own view?"); the user's typed responses are stored as journal annotations at the current CFI, and the source passages are shown in the context inspector, turning passive reading into active reflection without leaving the book and without any AI-generated answers displacing the reader's own thinking. Cost: M. Risk: low.
- **AI-assisted annotation tagging** — As a user writes a note, an AI call grounded in that note text and its source passage suggests 2–3 semantic tags (e.g. "irony", "counter-argument", "key term") that the user can accept or discard before saving; accepted tags are stored in the annotation model and make the full annotation collection filterable by reading concept without any bulk-processing step, complementing the retrospective cluster feature by tagging at creation time; the prompt assembly and tag parsing are pure, unit-testable transforms that reuse the existing passage-selection and annotation infrastructure with no new libraries. Cost: S. Risk: low.
- **First-impression annotation** — Before the AI analysis of a selected passage is displayed, the reader is offered an optional "your take first" text field whose response is saved as a typed annotation at the CFI; the AI reply then appears alongside that note so the reader's own interpretation is always primary and the model acts as a respondent rather than an authority; this directly enacts the CLAUDE.md principle that "AI should support reading, not dominate it" and is a pure UX flow over existing passage-selection and annotation infrastructure with no new libraries. Cost: S. Risk: low.
- **Steelman counter-argument** — When reading a non-fiction passage, the user can request the strongest possible objection to the author's argument grounded exclusively in the book's own retrieved chunks, encouraging dialectical critical thinking without leaving the passage; prompt assembly, chunk retrieval, and source-footer rendering reuse existing Phase 5 infrastructure as a pure transform, and every source passage used in the counter-argument is shown in the context inspector. Cost: S. Risk: low.
- **Analogical bridge** — When a reader encounters an abstract concept in a dense passage, they request an analogy grounded exclusively in examples, imagery, and language retrieved from elsewhere in the same book via the existing chunk index; the AI call is instructed to draw only from those retrieved passages so the clarification stays in the author's own world and vocabulary rather than importing foreign metaphors, and the full retrieved context is shown in the context inspector as a pure prompt-assembly transform over existing retrieval infrastructure with no new libraries. Cost: S. Risk: low.
- **Socratic dialogue mode** — A toggle in the chat panel switches AI responses from direct answers to book-grounded Socratic questions, prompting the reader toward their own insight rather than displacing their thinking; the mode is implemented as a single prompt-assembly branch that instructs the model to respond with a passage-grounded question instead of a conclusion, reuses the existing retrieval and context-inspector infrastructure with no new libraries, and directly enacts the CLAUDE.md principle that "AI should support reading, not dominate it." Cost: S. Risk: low.
- **Session reading contract** — At the start of a reading session, users optionally state an intention or question (e.g. "understand the author's stance on determinism"); at session end, an AI call grounded exclusively in highlights taken during that session reflects on how well those passages address the stated intention and surfaces one gap to explore next, turning each session into a purposeful loop of intent, reading, and reflection; implemented as a pure prompt-assembly transform over session-scoped annotations with every source passage shown in the context inspector. Cost: M. Risk: low.
- **Embedded citation extractor** — When a reader highlights a non-fiction passage where the author cites, names, or alludes to external works, studies, or thinkers, an AI call grounded exclusively in that passage and adjacent retrieved chunks extracts the implied references (title, author name, or study description) and stores them as a structured "reference" annotation type; this lets academic and research readers build a reading-embedded bibliography without leaving the book, reuses existing annotation and chunk-retrieval infrastructure as a pure extraction transform, and keeps the model anchored to the text with every source passage shown in the context inspector. Cost: S. Risk: low.
- **Unstated assumption surfacer** — When a reader highlights an argumentative passage, an AI call grounded exclusively in that passage and adjacent retrieved chunks identifies the unstated premises the argument silently takes for granted and renders them as a numbered list, with each premise linked back to the span from which it was inferred; making implicit reasoning visible is a core critical-reading skill not served by any existing feature (steelman argues against, Socratic mode asks questions, argument outline maps visible claims), and the feature is a pure prompt-assembly transform over existing chunk-retrieval and context-inspector infrastructure with no new libraries. Cost: S. Risk: low.
- **Tone shift detector** — An AI call processes the reader's highlights in reading order and identifies moments where the author's register markedly changes (e.g., analytical to impassioned, distant to intimate), producing a linked list of shift points each tied back to its source highlight in the context inspector; grounded exclusively in the user's curated selections, this is a pure prompt-assembly transform over the existing annotation model that surfaces authorial craft invisible to passive reading and complements theme clustering (topical patterns) and argument outline (structural claims) without overlapping either. Cost: M. Risk: low.
- **Foreshadowing revisit** — At any point a fiction reader chooses, an AI call grounded exclusively in their highlights from chapters already read identifies passages that appear in hindsight to foreshadow a narrative development the reader has since witnessed; this deepens appreciation of the author's craft for fiction readers, covers a use case unaddressed by any existing idea, and is a pure prompt-assembly transform over the existing spoiler-safe chunk-retrieval and CFI-fence infrastructure with no new libraries. Cost: M. Risk: low.
- **Authorial voice fingerprint** — A user-triggered AI call grounded exclusively in the reader's selected highlights characterizes the author's stable stylistic habits (e.g., favored rhetorical devices, sentence rhythm, reliance on qualification) as a plain-language profile stored as a book-level note; unlike tone shift detector this captures the baseline voice rather than tracking its changes, helping readers become consciously aware of how style shapes their reception of content and surfacing craft invisible to passive reading, with every source annotation shown in the context inspector as a pure prompt-assembly transform over the existing annotation model and no new libraries. Cost: S. Risk: low.
- **Argument chain tracer** — When a reader highlights a conclusion or claim, an AI call grounded exclusively in already-read chunks reconstructs the ordered chain of supporting steps the author builds toward it, rendering a passage-linked list from initial evidence to final conclusion; this helps analytical readers evaluate whether a conclusion is adequately supported, complements the top-down argument outline (book-level structure) and the unstated assumption surfacer (implicit premises) without duplicating either, and is a pure prompt-assembly transform over existing chunk-retrieval and context-inspector infrastructure with no new libraries. Cost: S. Risk: low.
- **Re-read discovery** — When a reader finishes (or revisits) a book, an AI call grounded in the full annotation set and a sample of early-chapter chunks identifies 3–5 opening passages that carry deeper meaning now that the full argument or narrative has unfolded (e.g. a thesis that only becomes precise after chapter 7, a scene whose irony is invisible on first pass); unlike foreshadowing revisit (fiction-only, craft-specific retrospective) this targets both fiction and non-fiction and frames output as forward guidance for a richer re-read, implemented as a pure prompt-assembly transform over existing annotation and chunk-retrieval infrastructure with every source passage shown in the context inspector. Cost: S. Risk: low.
- **Comprehension-weighted retrieval** — When a reader highlights a passage, an optional 1–3 tap "confidence rating" is stored as a lightweight field on the annotation; this local signal feeds into chunk-retrieval scoring so AI context assembly automatically up-weights passages the reader found challenging, letting comprehension difficulty steer the model's attention without any manual pinning step; the confidence score, retrieval weight, and selected chunks are all shown in the context inspector, keeping the feature fully transparent, and the weighting logic is a pure, unit-testable transform over the existing chunk index that is entirely distinct from semantic similarity or CFI-position filtering. Cost: M. Risk: low.
- **Comprehension gap review** — At any point, an AI call grounded exclusively in the user's chat-thread questions for a book (along with the source passages those questions were anchored in) audits the accumulated Q&A history to surface concepts the reader asked about repeatedly or that received partial answers, producing a short "still unclear" list and a "resolved" summary; this turns the chat log into an active comprehension audit that closes the loop between asking and learning, is a pure prompt-assembly transform over existing chat-thread and passage-retrieval storage with every source passage shown in the context inspector, and is entirely distinct from all annotation-based features since it treats the reader's own questions — not their highlights — as the evidence signal. Cost: S. Risk: low.
- **Rhetorical device spotter** — When a reader selects any passage, an AI call grounded exclusively in that text labels the rhetorical and literary devices at work (e.g., anaphora, irony, ethos appeal, euphemism) and briefly explains how each shapes meaning in context, helping readers develop writerly and critical awareness without leaving the book; this is a pure prompt-assembly transform over existing passage-selection infrastructure, requires no new libraries, and is entirely distinct from authorial voice fingerprint (stable baseline style), tone shift detector (register changes across the book), and unstated assumption surfacer (implicit logical premises). Cost: S. Risk: low.
- **Pre-chapter readiness brief** — Before the reader begins a new chapter, an optional AI call grounded in that chapter's opening chunks identifies 2–3 key concepts or background terms they will encounter and produces a "useful to know first" note, helping them decide whether to seek context before diving in; this is the only pre-reading orientation feature in the set (all other AI features respond after or during reading), is entirely distinct from in-context vocabulary lookup (single word during reading) and the reading-gap catch-up digest (returns to a book after absence), and is a pure prompt-assembly transform over the existing chunk-retrieval and chapter-navigation infrastructure with every source chunk shown in the context inspector and no new libraries. Cost: S. Risk: low.
- **Reader's editorial voice** — A user-triggered AI call grounded exclusively in the reader's complete annotation set characterizes their own analytical lens for a book — what they tend to notice, resist, and find profound — stored as a "your reading stance" note alongside the book; unlike annotation theme clustering (topical content patterns) and authorial voice fingerprint (the author's style), this turns the annotation collection into a self-reflective mirror on the reader's own interpretive biases and sensibilities, implemented as a pure prompt-assembly transform over the existing annotation model with every source annotation shown in the context inspector and no new libraries. Cost: S. Risk: low.
- **Pedagogical strategy decoder** — When a reader highlights a passage containing an example, analogy, or case study, an AI call grounded in that passage and adjacent retrieved chunks identifies the teaching technique the author is deploying (e.g., concrete-before-abstract, Socratic inversion, steelmanning-then-refutation) and briefly explains its function at that point in the argument; this surfaces how authors deliberately architect understanding — distinct from rhetorical device spotter (stylistic prose devices), authorial voice fingerprint (stable style baseline), and non-fiction argument outline (visible claim structure) — as a pure prompt-assembly transform with every source chunk shown in the context inspector and no new libraries. Cost: S. Risk: low.
- **Unreliable narrator detector** — For fiction readers who have accumulated highlights, an AI call grounded exclusively in those passages and adjacent retrieved chunks identifies signals of narrative unreliability (self-serving framing, inconsistencies across the narrator's account, evasiveness about key events) and lists each flagged signal with its source span so the reader can evaluate them critically; this distinct literary analysis use case is unaddressed by foreshadowing revisit (craft retrospective on known plot) or tone shift detector (register changes in the author's voice), and is a pure prompt-assembly transform over existing chunk-retrieval and annotation infrastructure with no new libraries. Cost: M. Risk: med.
- **Claim evidence strength labeler** — When a reader highlights an empirical or argumentative claim in a non-fiction passage, an AI call grounded in that passage and adjacent retrieved chunks assesses the type of support present (bare assertion, anecdote, cited study, or logical deduction) and stores a readable label on the annotation; this helps critical readers track which claims deserve provisional versus confident acceptance without leaving the text, is entirely distinct from steelman counter-argument (generates an opposing argument), unstated assumption surfacer (identifies hidden premises), and argument chain tracer (maps reasoning steps), and is a pure prompt-assembly transform over existing chunk-retrieval and annotation infrastructure with no new libraries. Cost: S. Risk: low.
- **Implicit value audit** — A user-triggered AI call grounded exclusively in the reader's accumulated highlights identifies the unstated values and normative assumptions the author brings to their subject (e.g., prioritizing efficiency over equity, treating certain outcomes as self-evidently desirable), surfacing ideological framing that operates below the level of individual arguments and is stored as a book-level note; distinct from unstated assumption surfacer (logical premises of specific claims), authorial voice fingerprint (stable stylistic habits), and reader's editorial voice (the reader's own interpretive biases), this is a pure prompt-assembly transform over the existing annotation model with every source annotation shown in the context inspector and no new libraries. Cost: S. Risk: low.
- **Epistemic confidence map** — A user-triggered AI call grounded in the reader's accumulated highlights and adjacent retrieved chunks identifies where the author hedges claims ("perhaps", "it may be that", "evidence suggests") versus speaks declaratively, producing a book-level note that tiers conclusions by the author's own expressed certainty; this helps critical readers distinguish confident conclusions from speculative ones and is entirely distinct from claim evidence strength labeler (quality of external supporting evidence), unstated assumption surfacer (logical premises), and implicit value audit (normative framing), implemented as a pure prompt-assembly transform over existing chunk-retrieval and annotation infrastructure with every source passage shown in the context inspector and no new libraries. Cost: S. Risk: low.
- **Marginal thought polisher** — When a reader types a rough, unfinished note (e.g. "this seems contradictory somehow?"), an AI call grounded in that draft and the source passage offers one sharpened phrasing the reader can accept or rewrite before saving, helping them articulate observations they sense but can't yet state precisely; this assists the underserved act of thinking-through-writing — left unaddressed by first-impression annotation (captures raw reaction before AI), AI-assisted annotation tagging (labels a finished note), and reflective journaling prompter (generates external prompts) — with the reader always in control and every source passage shown in the context inspector as a pure prompt-assembly transform over existing passage-selection and annotation infrastructure with no new libraries. Cost: S. Risk: low.
- **Internal contradiction spotter** — A user-triggered AI call grounded exclusively in the reader's accumulated highlights scans for pairs of passages that appear to assert mutually exclusive claims, presenting each conflicting pair with a brief explanation so the reader can judge whether the gap is a genuine inconsistency, an evolved position, or a deliberate unresolved tension; this consistency audit is entirely distinct from steelman counter-argument (invents an opposing argument), epistemic confidence map (tracks intentional hedging), and unstated assumption surfacer (identifies hidden premises per claim), and is a pure prompt-assembly transform over the existing annotation model with every source annotation shown in the context inspector and no new libraries. Cost: M. Risk: low.
- **Semantic drift tracker** — A reader names a key term used throughout the book (e.g. "reason", "justice", "freedom"), and an AI call grounded in their accumulated highlights retrieves every use of that term across already-read passages and notes how its meaning or emphasis shifts across the arc, presenting a chronological list of uses each linked to its source passage in the context inspector; this helps readers detect definitional shifts that quietly underpin an author's argument — a form of conceptual analysis unaddressed by tone shift detector (register changes in voice), epistemic confidence map (hedging vs. declarative confidence), and internal contradiction spotter (conflicting claims) — and is a pure prompt-assembly transform over existing chunk-retrieval and annotation infrastructure with no new libraries. Cost: S. Risk: low.
- **Cross-book argument structure comparison** — When a reader selects a shared topic across two books, an AI call grounded in retrieved chunks from both analyzes how each author structures their reasoning (inductive vs. deductive, case-study-led vs. principle-first, evidence-heavy vs. narrative-anchored) rather than comparing what they argue; this meta-rhetorical lens is entirely distinct from cross-book thematic comparison (open-ended user-posed content questions) and non-fiction argument outline (single-book claim hierarchy), helping analytical readers develop genre awareness of how non-fiction arguments are constructed, implemented as a pure prompt-assembly transform over the existing multi-book chunk-retrieval infrastructure with every source passage shown in the context inspector and no new libraries. Cost: M. Risk: low.
- **Conceptual metaphor tracer** — A user-triggered AI call grounded in the reader's accumulated highlights identifies the central organizing metaphors threading through the book (e.g. "argument is war", "knowledge is light") by clustering figurative language patterns across already-read passages into a book-level "conceptual vocabulary" note; unlike rhetorical device spotter (which labels devices per passage in isolation) or authorial voice fingerprint (surface-level stylistic habits), this surfaces the deep cognitive frames shaping how ideas are presented — a form of cross-passage pattern analysis that serves both literary and analytical readers — implemented as a pure prompt-assembly transform over the existing annotation model with every source passage shown in the context inspector and no new libraries. Cost: M. Risk: low.
- **Narrative distance tracker** — For fiction readers who have accumulated highlights, a user-triggered AI call grounded in those passages identifies how narrative distance shifts across the reading arc — from deep character interiority to detached omniscient observation — presenting a passage-linked list of perspective changes that helps readers understand how point-of-view manipulation shapes emotional experience and meaning; entirely distinct from unreliable narrator detector (narrator reliability), tone shift detector (authorial prose register), and rhetorical device spotter (surface devices per passage in isolation), this develops literary awareness as a pure prompt-assembly transform over existing annotation and chunk-retrieval infrastructure with no new libraries. Cost: M. Risk: low.
- **Character arc tracker** — For fiction readers, a user-triggered AI call grounded exclusively in highlights and adjacent retrieved chunks that mention a named character traces how that character's situation, motivations, and defining traits evolve across already-read passages, presenting a chronological passage-linked arc summary; unlike entity memory digest (a static "what we know" snapshot at a point in time), this focuses on the narrative of change to help readers consciously appreciate authorial craft in character development, and is a pure prompt-assembly transform over existing annotation and spoiler-safe chunk-retrieval infrastructure with no new libraries. Cost: S. Risk: low.
- **Emotional arc mapping** — A user-triggered AI call grounded exclusively in the reader's accumulated highlights maps the emotional register of each passage in reading order (e.g., tension, grief, wonder, unease, resolution), producing a passage-linked affective timeline that reveals the emotional shape of the reader's journey through the book; distinct from tone shift detector (which tracks the author's prose register), narrative distance tracker (POV shifts), and character arc tracker (character development), this surfaces what emotional content the reader was drawn to highlight rather than any craft choice of the author, and is a pure prompt-assembly transform over the existing annotation model with every source annotation shown in the context inspector and no new libraries. Cost: M. Risk: low.
